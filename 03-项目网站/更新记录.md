# 网站迭代记录

## 版本约定

- 当前采用轻量版本号：`v0.x`
- 一般迭代按 `0.1` 递增，例如：`v0.7 -> v0.8`
- 明显的阶段性升级、结构性重构或可对外宣讲的里程碑版本，再进入 `v1.0`
- 每次准备 push 到 GitHub 前，先补本文件，再提交代码与更新记录

## 当前版本

- 当前网站版本：`v0.8`
- 当前轮次的目标是继续向正式版收束，但本次不贸然标注为 `v1.0`

## v0.8 首页学术主线收束版

- 首页首屏改为以《葛覃》“害澣害否”为代表性实例开场，先讲一条考据怎样成立，再进入项目概述
- 首页文案进一步压低“平台展示 / Demo”气质，统一改回“研究整理原型”“考据过程”“案例链条”等口径
- 首页保留数据库统计与试查入口，但将申报时间线、团队分工收至底部折叠区，减少对学术主线的打断
- 二级页标题、徽标与页脚同步去除过重的 `Demo` 表述，统一为“研究原型”口径
- 已同步检查 `01-项目申报/立项申报v4.1.md` 与 `01-项目申报/更新记录.md`，确认首页改写仍与申报书主线一致

## v0.1 原型展示版

- 完成首页、项目概述、时间线、团队结构等基础静态展示
- 建立最小必要 5 表的展示思路
- 以 demo 数据为主，强调“能跑通”的项目原型

## v0.2 工程化接入版

- 重构 Node 服务入口，统一页面分发与 API
- 新增 SQLite 快照桥接，接入 `大创` 实库数据
- Railway 部署链路稳定，默认可直接启动

## v0.3 检索重构版

- 首页从“只看词条群”改为“字词优先 + 相关案例”
- 新增统一 `/api/search`
- 案例标题改为更适合展示的格式，如 `《广雅疏证》「始也」条`

## v0.4 详情页版

- 新增字词详情页 `term.html`
- 新增案例详情页 `case.html`
- 详情页补充证据、关联案例、原始字段，并使用折叠降低信息压力

## v0.5 数据库浏览版

- `database.html` 从架构说明页升级为数据库浏览页
- 支持左侧类别切换、右侧列表浏览与检索
- 搜索方式分为：
  - 词条/标题检索
  - 正文/全文检索
- 架构说明并入数据库浏览页内部，不再单独割裂

## v0.6 首页轻量化与展示优化版

- 首页字词与案例固定只展示 6 个入口
- “阅读全部”跳转到数据库浏览页，首页不再承载完整浏览任务
- 首页首屏改为展示型 Hero，突出数据库入口、案例入口与代表性案例
- 检字流程更新为图片版 `media/step.png`

## v0.7 展示站与数据库浏览整合版

- 首页改为“展示首屏 + 下滑看介绍”的结构，强化项目气质与快速入口
- 新增案例库快速入口、数据库浏览入口、代表性案例入口
- `database.html` 正式承担数据库浏览任务，不再只是架构说明页
- 数据库浏览页支持：
  - 左侧浏览类别
  - 左侧分类筛选
  - 右侧结果浏览
  - 词条/标题检索
  - 正文/全文检索
- 数据库浏览页加入左栏空态保护与稳态重绘，避免切换时出现空白侧栏
- 首页流程图替换为最新图片资源，适配投屏与网页展示

## 后续方向摘要

- 将案例浏览页进一步改成更轻的档案列表视图
- 为数据库浏览页增加更明确的空状态、加载状态与分页
- 如果后续案例分类变丰富，再补多级类别树和筛选标签
