# 网站迭代记录

## 版本约定

- 当前采用轻量版本号：`v0.x`
- 一般迭代按 `0.1` 递增，例如：`v0.7 -> v0.8`
- 明显的阶段性升级、结构性重构或可对外宣讲的里程碑版本，再进入 `v1.0`
- 每次准备 push 到 GitHub 前，先补本文件，再提交代码与更新记录

## `2026-04-07` 与申报书 `v4.2` 联动核查

- 需要记住的边界是：网站首页已经以“造舟于河”显式展示了“发疑 - 设问 - 取证 - 释理 - 结论”的链条，但这一结构目前仍是首页代表性案例的展示方式，尚未推广为所有案例详情页的统一字段结构；后续若全站进入这一层系统化展示，再单独记一轮版本更新。
- 同日又将首页旧的“检字流程”图片替换为 `v4.2` 释读架构图，明确区分“首页个案五环节展示”和“更上位的方法架构图”两层内容，避免把个案展示链误写成总体释读结构。

- 申报书侧随后又补入两处固定图片占位，分别对应首页展示图和“造舟于河”案例结构图；这些占位与图注都直接对应网站现有页面内容，后续答辩时可直接截图使用。
