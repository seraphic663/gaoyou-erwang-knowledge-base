# 网站反馈：人文导向调整

## 1. 调整目的

本轮网站修改的目标，不是继续强化“平台功能感”，而是把首页与二级页的叙事顺序调整为：

学术问题优先 -> 材料与整理单位 -> 最小必要数据库 -> 案例浏览与展示

对应到项目正式口径，就是让网站更清楚地服务于“高邮二王考据过程知识库构建及应用”，而不是把“数据库浏览”“可视化展示”“平台功能”写成主角。

## 2. 主要参考来源

本次改动主要参考了 `01-项目申报` 下几份已经较稳定的 Markdown 文稿：

- `01-项目申报/立项申报v3.3.md`
- `01-项目申报/立项申报核心提炼.md`
- `02-研究方案/数据库任务与字段规范.md`

其中最值得直接吸收的判断有三条：

1. 项目对象应稳定写成“高邮二王”及“王氏四种”，不能泛化为一般古籍平台。
2. 网站要优先呈现“考据过程”“考据案例”“证据链条”，不能只展示数据库和检索能力。
3. 最小必要数据库的意义，不是结构复杂，而是能够保留“著作、文本片段、词条、案例、证据”之间的关系。

## 3. 当前网站最需要收口的地方

修改前，首页仍有几处明显偏“技术主轴”的表述：

- 标题写成“高邮二王知识库构建及应用”，但没有突出“考据过程”。
- 首屏副文案强调“可检索、可解释、可视化的古典知识平台”，平台感过强。
- “项目概述”部分过于扁平，缺少“研究问题”“材料边界”“整理单位”。
- “最小必要数据库概览”更像在讲架构，而不是解释为什么必须这样组织材料。
- 首页虽有案例入口，但还没有明确回答“一条案例如何成立”。
- 二级页面仍以“高邮二王知识库”为品牌名，与正式申报口径不完全一致。

## 4. 建议的网站叙事顺序

首页建议固定为以下顺序：

1. 研究对象与研究问题
2. 为什么现有平台还不够
3. 为什么要用最小必要数据库
4. 一条案例如何被串成证据链
5. 再进入字词与案例试查

这个顺序的好处是，评委或老师一打开页面，先看到的是“你在研究什么”，而不是“你做了多少功能”。

## 5. 本轮实际修改原则

### 5.1 术语统一

- 统一使用“高邮二王考据过程知识库”
- 正式标题统一为“高邮二王考据过程知识库构建及应用”
- 少用“古典知识平台”“可视化平台”“知识图谱驱动”等空泛说法
- 多用“考据过程”“考据案例”“证据链条”“最小必要数据库”“双向检索”

### 5.2 首页改写方向

- 首屏先讲研究问题，而不是入口聚合
- “项目概述”改为“研究问题与对象”
- “数据库概览”改为“为什么要做最小必要数据库”
- 新增“一条案例如何成立”模块
- 检索区改成“从字词或问题进入案例”，弱化单纯搜索框气质

### 5.3 二级页改写方向

- 数据库页文案改为“整理结果浏览”，不单讲架构
- 字词页、案例页统一品牌和页脚
- “数据库架构”一类表述改为“数据库浏览”或“整理结构”

## 6. 后续还可继续补的内容

如果继续迭代网站，最值得优先补的不是更多特效，而是下面三类内容：

1. 代表性案例的成链说明
2. 同类问题的并排比较
3. 术语、方法、证据类型之间的说明性文字

只要网站能更清楚回答“这个系统帮助我们怎样回到一条具体考据”，整体气质就会比一般展示型 Demo 更稳。
