# 高邮二王考据过程知识库构建及应用

## 一、项目简介

**【说明】** 对项目研究的摘要。限500字，含标点符号。

高邮王念孙、王引之父子及其“**王氏四种**”是清代训诂学的重要成果，其价值不仅在于留下了大量释读结论，更在于形成了一套围绕经文疑难展开判断的**考据过程**：从何处发疑、如何取证、怎样比较字形音义、最终如何达成解释。对古汉语、训诂学和音韵学研究而言，二王的重要性不仅在“说了什么”，更在“怎样判断”，尤其体现在通假、因声求义、联绵词、虚词释读、句法辨析等疑难问题的处理方式上。

当前研究者读到二王的一条考释时，往往要反复完成几步工作：先回查二王原文，再补出所涉经文，接着核对引证材料，最后才有可能看清这条解释是怎样成立的。现有古籍平台能帮人较快找到文本，却难以把这几步连成一条清楚的判断链。

本项目拟以“王氏四种”为核心材料，围绕二王考据中的典型问题类型，逐条整理代表性案例：先摘出二王论述，再补齐原始经典和引证材料，接着标明问题类型、用证顺序和结论，最后组织成可检索、可复核的专题知识库。技术手段在这里不是单独要炫示的“平台功能”，而是帮助我们把原本分散的材料连起来、把原本隐在文字背后的判断过程呈现出来。项目目标不是替代学术判断，而是为二王方法研究、古籍教学与后续专题整理提供一套更省力、更清楚的整理办法。

## 三、项目立项依据与文献综述

**【说明】** 包括研究意义、国内外研究现状和趋势综述、参考文献和其他有关背景材料等，应通过文献综述凸显本研究的必要性与学术贡献。除参考文献列表外限3000字，含标点符号。

### 1. 研究意义

本项目的意义主要体现在两个层面。

第一，二王是清代训诂学与校读学的重要代表，其成果不仅体现在对若干字词的释读结论上，更体现在一套可被反复运用的方法论上。若只保存结论而不保存过程，二王研究很容易停留在“个案摘录”或“思想概述”层面，难以进一步转化为可比较、可积累、可迁移的学术资源。围绕“王氏四种”开展结构化整理，有助于把分散于条目、札记和引文中的判断链条重新组织起来，并进一步观察二王在通假、因声求义、联绵词、虚词释读、句法辨析等问题上的常见路径，为二王研究、清代学术史研究和古典文献学教学提供新的材料基础。

第二，从数字人文的角度看，古籍数字化的难点已经不只是“把文本放到网上”，而是如何把学术工作中的对象、步骤、证据和判断关系表达出来。就古籍研究而言，很多真正耗时的工作并非找到文本，而是辨析疑难、归并同类、核对书证和复原解释路径。本项目所尝试建设的，不是一个泛化的古籍平台，而是一个针对具体学术对象、能够保留过程信息的知识库。其价值在于为传统学术资源建立可持续积累的整理方式，也为后续同类专题的数字化整理提供一个可以复制的方法样本。

### 2. 相关研究与现状

就目前团队已阅读和整理的材料看，与本项目直接相关的研究大致可以分为三类。

第一类是围绕高邮二王及“**王氏四种**”的传统研究。既有成果已经较充分地揭示了二王在训诂、校勘、虚词研究和经学解释中的学术地位，也积累了大量个案分析与思想史论述。这类研究奠定了本项目的学术基础，使我们能够明确二王治学的核心问题、典型方法和文本重心。但现有研究多以专题论文、个案阐释或著作评述的形式展开，成果分散，较少将二王的论证过程作为一个整体对象加以系统编码和比较。

第二类是古籍数字化、数字人文与知识组织方法相关研究。近年来，全文检索、影像浏览、点校文本查询等基础能力已有明显进展，数字人文方法也逐步进入古典文献研究领域。贾楠、龚蛟腾的综述偏重整体梳理，不只讨论“古籍数字化”这一笼统方向，而是具体归纳了可视化挖掘、智能化处理、专类化挖掘、古典文献学赋能和用户深层需求等数个热点，同时指出相关研究仍主要集中于技术研究和平台建设，与历史、文学、语言学、哲学等人文学科的紧密结合尚显不足。王军等人的系列笔谈则更像一种立场提醒：文中一方面承认 OCR、自动标点、命名实体识别、知识库等工具正在改变古籍整理方式，另一方面又反复强调古典文献学不能脱离文献本体，版本、目录、校勘以及文字、音韵、训诂等传统训练仍是根基，技术与文献学的结合不能只是简单拼接。杨浩的文章进一步把当前相关研究上升到方法论层面，提出算法驱动的互文性研究、AI赋能的文本分析研究、基于大模型的数据抽取三种模式，显示数智时代的古典文献研究正在由一般检索走向更细的结构化分析与抽取。

综合来看，贾楠、龚蛟腾更像是在说明这个领域整体走到了哪里，并提醒古籍数字人文研究不能只停在技术和平台层面；王军等人更强调在引入新技术时，传统文献学根基与人文尺度不可丢；杨浩则把当前最活跃的技术路径归纳为三种较有代表性的模式。三者共同说明：古籍研究当然可以进入数字整理，但现有讨论多停留在宏观趋势、平台生态或通用方法层面，对一条具体考据如何从发疑、取证、比较到形成解释这一更小、更细的过程单元，仍缺少贴近文献学实践的专题整理。

第三类是面向专题知识库的轻量原型实践。相比成熟商用平台，本类工作更强调“先建立可验证的专题模型，再逐步扩展”。就本项目自身进展而言，团队已经围绕高邮二王知识库搭建了本地 Demo，初步实现了案例浏览、基础检索、数据库结构说明和演示页面组织。

这一原型并不意味着项目已经完成，相反，它的意义在于验证了一个关键判断：二王材料并非只能以传统札记形式存在，而是可以被拆成若干清楚的整理单元，并按“著作 - 片段 - 案例 - 证据”的顺序重新组织。换言之，本项目并非从零开始想象一个平台，而是在已有试作基础上继续推进学术化与规范化整理。

### 3. 本项目的必要性与拟补缺口

本项目的必要性，不在于重复建设一个新的全文检索平台，而在于补足现有研究与工具之间的空白地带：今天关于二王的材料并不少，真正困难的是把一条考释重新走通。研究者常常要自己完成“找二王原文 - 回到所涉经文 - 核对引书和旧说 - 判断二王为何取此说”这几步，而这几步往往分散在不同书、不同页、不同笔记中。本项目要做的，就是把这条动作链先替研究者整理出来，再把整理结果组织成可以继续查、继续比、继续补的案例库。

更具体地说，项目拟补的缺口至少包括三层。第一，二王论述、原始经典和引证材料长期分散，查找入口不统一。第二，即便找到相关条目，研究者也很难迅速看清这条判断是怎样一步步成立的。第三，现有成果多保存在论文、札记和批注中，适合细读，却不利于把同类案例放在一起比较。为了解决这三个问题，本项目将先做案例整理，再做入库和展示，使材料、判断过程和比较路径能够放在同一套工作流中。

与上述研究相比，本项目的优势不在于平台更大或算法更复杂，而在于对象更聚焦、问题更具体、学术约束更强。项目以“王氏四种”为明确边界，以“单条考据案例”为基本整理单位，不直接处理海量古籍的一般性抽取任务，而是重点保存二王如何发疑、怎样取证、怎样比较字形音义并达成解释；同时坚持人工复核、术语审核和文献依据优先，不把自动抽取或智能分析预设为前提。这种做法更接近训诂学和校读学的实际工作现场，也更符合古典文献研究对可解释性和可复核性的要求。

### 4. 研究趋势判断

从趋势上看，古籍数字化正在从“资料上网”转向“知识组织”，数字人文也在从通用平台建设转向更有学科针对性的专题资源建设。对于训诂学、校读学这类高度依赖论证过程的领域而言，下一步更值得推进的，显然不是再多做一个检索页面，而是把问题意识、证据使用和判断路径保存下来。更进一步说，这一趋势并不意味着技术展示本身成为目标，而是要求专题资源建设同时回应古典文献研究中的人文学术问题、知识服务需求与协同共享可能；只有在保持文献学审慎和人工校核前提下，数字工具的价值才真正能够成立。本项目正是顺着这一趋势，选择一个边界清楚、材料集中、方法特征鲜明的对象开展试验。其贡献不在于一次性完成所有智能化目标，而在于证明传统考据过程可以被比较稳妥地整理、标注和展示，为进一步研究打下基础。

从实施方式看，专题数据库与轻量网站原型结合也符合当前数字人文项目的发展路径。一方面，数据库负责保证材料组织的稳定性和学术可追溯性；另一方面，网站原型负责把研究成果转化为可浏览、可交流、可持续补充的界面。对本科项目而言，这种路径比空泛讨论“智能系统”更稳健，也更容易形成清楚的阶段性成果链条。

## 四、项目研究方案

**【说明】** 包括研究目标、研究内容、研究方法、可行性分析等，应围绕一个明确且具有学术价值的研究问题，精述将如何开展本研究。限4000字，含标点符号。

### 1. 研究问题

本项目拟围绕一个明确问题展开：高邮二王在古籍校读与训诂中的判断过程，能否在不削弱学术复杂性的前提下，被整理成可检索、可复核、可持续扩展的数字资源？更进一步说，本项目希望回答的并不是“能否做一个网站”，而是“二王在古汉语、训诂学、音韵学层面最有价值的过程性知识，能否被稳妥地保存下来，并转化为可比较、可复核的研究材料”。

这一问题可以拆成四个子问题：

1. 二王的单条考据应当以什么粒度作为整理单位。
2. 在通假、因声求义、联绵词、虚词释读、句法辨析等类型中，二王常用的判断路径有哪些异同。
3. 考据过程中的“问题、方法、证据、结论”如何稳定表达，并与原始经典、引证材料形成可追溯关联。
4. 在一年期本科项目中，哪些部分适合先做成稳定成果，哪些部分只适合做探索性延伸。

### 2. 研究目标

本项目的总体目标是：围绕“**王氏四种**”建立一套能够保存二王**考据过程**、支持基础检索与案例展示、并可继续扩展的专题知识库原型，同时据此对二王处理典型疑难问题的方法类型形成初步归纳。

在此基础上，项目拟完成以下几项可验证目标：

1. 从“王氏四种”中选出首批代表性案例，补齐出处、原文和引证材料。
2. 为每条案例建立统一的记录格式，至少写清问题类型、主要证据、判断过程和结论。
3. 把这些案例做成一个本地可运行的演示原型，支持基础查找和案例浏览。
4. 在首批案例基础上，归纳二王处理几类典型疑难问题时常用的方法和用证方式。
5. 形成一套后续可以继续补案例、继续修订的工作规则。

### 3. 研究内容

本项目的研究内容主要包括五部分。

第一，文本与材料整理。团队将以“王氏四种”为核心材料，优先整理与方法特征较鲜明的条目、札记和代表性案例，同时补录其所涉及的原始经典片段和引证材料。材料整理时将重点保留出处信息，尽量做到可以逐级回溯至卷次、篇章、条目或其他可识别位置。

第二，结构化标注与案例抽取。结合前期讨论，项目将“单条考据”作为基本工作单元，围绕问题触发点、主要方法、证据来源、过程说明、暂定结论等维度进行标注。对于当前阶段不宜过度细分的内容，先采用整体记录方式保存，避免为了形式完整而提前设计过重结构。在案例组织上，项目将尽量围绕若干典型问题类型形成小规模案例组，以便观察同类疑难在不同文本中的处理方式。

第三，知识库与演示系统实现。依据现有数据库设计方案，当前阶段先把著作、文本片段、案例和证据这几类材料接起来，完成录入、查找和展示的基本闭环。项目组已形成对应的字段规范与本地 Demo 原型，后续将在此基础上继续补充数据和完善交互，使不同阶段的研究进展都能有相应的呈现方式。这里的技术实现主要承担材料承载、案例组织与研究展示功能，而不作为独立于学术问题之外的目标。

第四，方法归纳与学术分析。在案例累积到一定规模后，团队将把同类问题放在一起比较，例如二王遇到通假问题时通常先看什么，处理联绵词时更依赖哪些证据，解释虚词时如何兼顾语境和旧说。这一部分既是展示内容的基础，更是本项目在古汉语、训诂学、音韵学层面希望形成的阶段性认识。

第五，研究成果的前端化表达。项目不把网站视为独立于研究之外的“包装层”，而是把它作为研究成果表达的一部分。后续将通过案例浏览、字段说明和基础检索，把一条案例是怎样被整理出来的、又怎样与其他案例连起来，较清楚地呈现出来。对本科项目而言，这种可展示成果能够提高研究工作的可检验性，但其服务对象仍然是二王考据方法的整理与比较，而不是界面本身。

### 4. 研究方法

本项目采用文献学方法与数字整理方法结合的方式推进。

1. 文献细读与个案分析。对代表性条目逐条研读，先看二王在解释什么，再看他怎样展开判断。
2. 训诂与音义关系分析。围绕通假、因声求义、联绵词、虚词释读等问题，结合字形、音读、语境和书证做细读比较。
3. 结构化标注。把每条考据拆成若干固定栏目，至少记录原文、问题、证据、过程和结论。
4. 分类归纳。把同类问题放在一起看，比较它们的共性与差异。
5. 数据建模。把已整理的著作、片段、案例和证据按稳定关系接起来，保证后续能查、能比、能补。
6. 原型开发。把已整理的数据做成可运行的本地原型，用实际使用过程反过来修正字段和流程。
7. 人工复核优先。任何自动处理或程序辅助都只作为整理工具，不替代最终学术判断；涉及疑难归类和结论确认时，以人工核校为准。

### 5. 技术路线与实施步骤

结合当前项目基础，实施步骤拟分四阶段推进。

#### 阶段 A：数据准备与规则确定

1. 清理现有 Word、Markdown 等材料。
2. 统一案例记录格式、出处标记方式和术语口径。
3. 明确当前阶段采用的最小必要数据库结构。

#### 阶段 B：基础语料录入与检索原型

1. 录入首批代表性著作信息、文本片段和案例数据。
2. 建立词条、案例、证据之间的基本连接关系。
3. 完成知识库 Demo 中的基础检索、统计和案例浏览功能。
4. 形成与研究内容相对应的基础展示层次，确保案例、出处与方法说明能够清楚对应。

#### 阶段 C：案例扩充与方法归纳

1. 扩充案例数量，覆盖若干典型问题类型。
2. 对常见方法路径进行分类整理。
3. 结合数据库中已录入的案例、证据和术语，形成阶段性研究说明和标注规范说明。
4. 进一步优化网站展示，使案例、出处、证据和方法标签能够形成相互印证的阅读路径。

#### 阶段 D：探索性扩展

1. 在已有数据基础上试做更细关系或规则表示。
2. 讨论是否进入更复杂的关联结构或知识图谱表达。
3. 这一阶段仅作为探索性延伸，不设过高刚性指标。

### 6. 可行性分析

本项目具备较好的实施基础。

其一，材料边界清楚。项目以“王氏四种”为核心，不做无边界扩展，便于在一年周期内集中力量完成较高质量的专题整理。

其二，团队分工明确。学术整理、技术实现和数据处理已有对应成员承担，前期讨论也已经形成了较清楚的任务链条。

其三，已有原型基础。项目组已完成数据库字段规范、最小必要结构评估和本地 Demo 原型，原型中已包含基础案例数据、结构说明和展示逻辑。这意味着系统实现不是空想，而是已有可继续迭代的基础工程，也说明团队对“学术对象如何转化为可展示数据”已有初步实践。

其四，目标设置克制。项目不承诺当前无法证明的准确率、推荐效果或自动判读水平，不把“智能推理”写成必须完成的刚性成果，而是将重点放在文本整理、案例结构化、数据库搭建和工具原型建设上。这种安排更符合本科项目周期，也更有利于形成清楚、稳定的阶段性成果。

### 7. 风险与应对

1. 文本质量不一。部分材料可能存在 OCR 误差或版本差异。对此，项目将优先保留出处信息和原文片段，逐步精修，不以一次录入即达到最终标准为前提。
2. 结论存在多解。古籍解释并非都能形成唯一答案。项目将通过“暂定结论”“待核状态”等方式记录不确定性，而不强行求单解。
3. 工作量可能超出预期。对此将坚持“先完成最小闭环，再考虑扩展”的原则，优先保证 A-C 阶段成果。

## 五、项目特色与创新之处

**【说明】** 包括如何突破现有研究难点，提出创新的理论视角或实践应用或技术改进等，以体现本研究独特的学术贡献与创新价值。限1000字，含标点符号。

本项目的特色与创新主要体现在以下几个方面。

第一，项目切入角度明确。项目不泛泛而谈“古籍数字化”，而是围绕高邮二王这一方法特征鲜明、材料相对集中、学术影响明确的对象展开，尝试把以往分散在条目、札记和个案分析中的**考据过程**整理为可检索、可展示、可比较的专题资源。就当前可见的二王研究与古籍数字化实践而言，此类以考据过程为核心的专题整理仍较少见，这也是本项目最主要的辨识度所在。

第二，项目的创新重点不只在“能检索”，更在“能比较”。我们不是把零散材料简单堆到一起，而是准备把同类问题放在一起看：例如二王处理通假时先看什么，处理联绵词时更重哪类证据，解释虚词时如何兼顾上下文和旧说。这样做的结果，不是多一个功能标签，而是让以往分散的考据经验第一次能够被并排阅读和比较。

第三，项目拟设置“**二王校读流程**”模块，突出过程展示。对于研究者和初学者而言，最难把握的往往不是结论，而是为何会这样判断。本项目将尝试围绕单条案例，把这条解释拆成几个清楚步骤：问题从哪里起、证据先看什么、不同材料如何比较、最后为何落到这一结论。该模块的意义不在于展示界面本身，而在于把二王处理音义关系、通假和语境疑难的思路更清楚地呈现出来。

第四，数据库设计直接服从学术问题，而不是停留在技术口号层面。项目当前采用的**最小必要数据库**，把著作、文本片段、词条、案例、证据作为核心对象，其重点不是追求结构复杂，而是保证“二王说了什么、讨论了什么原文、如何展开论证、用了哪些证据”能够被清楚表达，并为后续比较不同问题类型中的判断路径提供材料基础。对训诂学专题数据库而言，这种以学术问题反推数据结构的做法，本身就是方法上的创新。

第五，项目在呈现形式上强调**双向检索**而非单向摘录。研究者可以先从某个字词出发，回看它在经典和二王著作中的不同解释；也可以先从某条经典或某个案例出发，再追到相关字词、证据和方法术语。这样的设计不是为了增加页面功能，而是为了减少研究者在不同材料之间来回翻找的时间。

第六，项目成果兼具学术研究、教学转化与传播价值。对研究者而言，它能减少重复检索，并提供首批按问题类型整理好的案例；对教学而言，它能把“怎么读懂一条考据”这件事拆给学生看；对项目实施而言，它也能把已整理案例、已归纳方法和阶段性成果较清楚地呈现出来，避免研究进展只停留在笼统说明上。
