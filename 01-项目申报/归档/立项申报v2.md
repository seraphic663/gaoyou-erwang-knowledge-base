# 高邮二王考据过程知识库构建及应用

## 一、项目简介

**【说明】** 对项目研究的摘要。限500字，含标点符号。

高邮王念孙、王引之父子及其“王氏四种”是清代训诂学的重要成果，其价值不仅在于留下了大量释读结论，更在于形成了一套围绕经文疑难展开判断的考据路径：从何处发疑、如何取证、怎样比较字形音义、最终如何达成解释。当前古籍数字平台在全文检索、影像浏览和文本查询方面已有明显进展，但对于“解释为何成立”这一过程层知识支持不足，研究者往往仍需在分散札记、经典原文和引证材料之间反复检索、比勘和归纳。本项目拟以“王氏四种”为核心材料，建设一个面向研究与教学使用的高邮二王考据过程知识库，将二王论述文本、所涉经典原文、关键词条、考据案例与证据材料进行结构化整理，形成可检索、可追溯、可复核的数字资源。项目当前采取克制而务实的实施策略：先完成基础语料整理、案例标注规则、最小必要数据库和演示原型，再在数据积累基础上讨论更复杂的关联结构。项目目标不是替代学术判断，而是为古籍研究提供一个能够保存论证过程、减少重复检索并支持持续积累的基础工具。

## 三、项目立项依据与文献综述

**【说明】** 包括研究意义、国内外研究现状和趋势综述、参考文献和其他有关背景材料等，应通过文献综述凸显本研究的必要性与学术贡献。除参考文献列表外限3000字，含标点符号。

### 1. 研究意义

本项目的意义主要体现在两个层面。

第一，二王是清代训诂学与校读学的重要代表，其成果不仅体现在对若干字词的释读结论上，更体现在一套可被反复运用的方法论上。若只保存结论而不保存过程，二王研究很容易停留在“个案摘录”或“思想概述”层面，难以进一步转化为可比较、可积累、可迁移的学术资源。围绕“王氏四种”开展结构化整理，有助于把分散于条目、札记和引文中的判断链条重新组织起来，为二王研究、清代学术史研究和古典文献学教学提供新的材料基础。

第二，从数字人文的角度看，古籍数字化的难点已经不只是“把文本放到网上”，而是如何把学术工作中的对象、步骤、证据和判断关系表达出来。就古籍研究而言，很多真正耗时的工作并非找到文本，而是辨析疑难、归并同类、核对书证和复原解释路径。本项目所尝试建设的，不是一个泛化的古籍平台，而是一个针对具体学术对象、能够保留过程信息的知识库。其价值在于为传统学术资源建立可持续积累的整理方式，也为后续同类专题的数字化整理提供一个可以复制的方法样本。

### 2. 相关研究与现状

就目前团队已阅读和整理的材料看，与本项目直接相关的研究大致可以分为三类。

第一类是围绕高邮二王及“王氏四种”的传统研究。既有成果已经较充分地揭示了二王在训诂、校勘、虚词研究和经学解释中的学术地位，也积累了大量个案分析与思想史论述。这类研究奠定了本项目的学术基础，使我们能够明确二王治学的核心问题、典型方法和文本重心。但现有研究多以专题论文、个案阐释或著作评述的形式展开，成果分散，较少将二王的论证过程作为一个整体对象加以系统编码和比较。

第二类是古籍数字化与在线平台建设。近年来，全文检索、影像浏览、点校文本查询等基础能力已有明显进展，这些平台极大提高了查找原文、比对异文和回溯出处的效率。然而，这类平台的重心主要仍是“文本定位”，对“解释过程”支持有限。研究者可以较快找到“某字在哪里出现”，却仍难直接追踪“二王为何在此作此判断、调用了哪些书证、使用了何种术语和路径”。换言之，现有平台更适合提供对象层知识，而对方法层知识的组织仍不充分。

第三类是数字人文与知识组织方法在文史研究中的应用。结构化标注、关系抽取、知识组织和可视化方法已在人物、地名、书目、文献传播等领域显示出价值，说明人文研究中的复杂对象并非不能被结构化表达。但就目前可见成果而言，针对清代训诂学者具体考据过程的数字化复原仍较少，尤其缺少同时兼顾原文定位、问题类型、证据链条和术语分类的专题数据库。这正是本项目准备进入的空档。

第四类是面向研究展示的轻量原型实践。相比成熟商用平台，本类工作更强调“先建立可验证的专题模型，再逐步扩展”。就本项目自身进展而言，团队已经围绕高邮二王知识库搭建了本地 Demo，初步实现了案例浏览、基础检索、数据库结构说明和演示页面组织。这一原型并不意味着项目已经完成，相反，它的意义在于验证了一个关键判断：二王材料并非只能以传统札记形式存在，而是可以被整理为著作、文本片段、词条、案例、证据等相互关联的数据对象，并在网页端被稳定展示出来。换言之，本项目并非从零开始想象一个平台，而是在已有试作基础上继续推进学术化与规范化整理。

### 3. 本项目的必要性与拟补缺口

本项目的必要性，不在于重复建设一个新的全文检索平台，而在于补足现有研究与工具之间的空白地带：一方面，已有二王研究为我们提供了丰富的对象知识；另一方面，现有数字平台提供了越来越成熟的文本检索环境；但二王考据过程本身尚未被系统整理成可检索、可追溯、可复核的结构化资源。本项目拟以“文本对象 + 考据案例 + 证据材料 + 方法术语”为基本单元，先建立最小必要数据库，再逐步扩展关联结构，使研究者能够从单条案例进入，也能够从字词、术语、文献来源或证据类型反向检索。换句话说，本项目真正要补的，不是“有没有文本”，而是“文本之间有没有被组织成可用的学术知识”。

更具体地说，项目拟补的缺口至少包括三层。第一层是材料层缺口，即二王论述、原始经典和引证材料长期分散在不同文本载体中，缺少统一的专题整理入口。第二层是过程层缺口，即即便研究者已经找到相关条目，也很难快速把“发疑何在、证据为何、结论如何成立”这一过程重新拼接出来。第三层是展示层缺口，即传统研究成果往往以论文、札记、批注方式保存，适合细读，却不利于横向比较、条件筛选和阶段性演示。本项目将数据库设计与网站展示同时纳入方案，正是希望在材料、过程和展示三个层面同步补足这一缺口。

### 4. 研究趋势判断

从趋势上看，古籍数字化正在从“资料上网”转向“知识组织”，数字人文也在从通用平台建设转向更有学科针对性的专题资源建设。对于训诂学、校读学这类高度依赖论证过程的领域而言，下一步更值得推进的，显然不是再多做一个检索页面，而是把问题意识、证据使用和判断路径保存下来。本项目正是顺着这一趋势，选择一个边界清楚、材料集中、方法特征鲜明的对象开展试验。其贡献不在于一次性完成所有智能化目标，而在于证明传统考据过程可以被比较稳妥地整理、标注和展示，为进一步研究打下基础。

从实施方式看，专题数据库与轻量网站原型结合也符合当前数字人文项目的发展路径。一方面，数据库负责保证材料组织的稳定性和学术可追溯性；另一方面，网站原型负责把研究成果转化为可浏览、可汇报、可交流的界面。对本科项目而言，这种路径比空泛讨论“智能系统”更稳健，也更容易形成能在申报、中期和结项三个阶段持续展示的成果链条。

## 四、项目研究方案

**【说明】** 包括研究目标、研究内容、研究方法、可行性分析等，应围绕一个明确且具有学术价值的研究问题，精述将如何开展本研究。限4000字，含标点符号。

### 1. 研究问题

本项目拟围绕一个明确问题展开：高邮二王在古籍校读与训诂中的判断过程，能否在不削弱学术复杂性的前提下，被整理成可检索、可复核、可持续扩展的数字资源？更进一步说，本项目希望回答的并不是“能否做一个网站”，而是“传统考据中最有价值的过程性知识，能否被稳妥地保存下来并真正服务后续研究”。

这一问题可以拆成四个子问题：

1. 二王的单条考据应当以什么粒度作为整理单位。
2. 考据过程中的“问题、方法、证据、结论”如何稳定表达。
3. 二王论述与原始经典、引证材料之间应如何建立可追溯关联。
4. 在一年期本科项目中，哪些部分适合先做成稳定成果，哪些部分只适合做探索性延伸。

### 2. 研究目标

本项目不追求在一年内完成“大而全”的知识图谱系统，而拟完成以下几项可验证目标：

1. 以“王氏四种”为核心，建立首批可用的结构化考据案例语料。
2. 建立一套适合当前阶段的知识库字段规范，明确著作、文本片段、词条、案例、证据之间的关系。
3. 开发一个本地可运行的知识库 Demo，实现基础检索、案例展示和数据库结构说明。
4. 在案例整理基础上，归纳二王校读学中较常见的问题类型、方法路径和证据使用方式。
5. 形成可供后续继续扩充的工作规范，为第二阶段扩展关系结构和规则总结预留接口。

这组目标的特点是：每一项都能对应到具体产出，而不是停留在概念承诺。项目完成时，评审既可以看到文本整理成果，也可以看到数据库结构和演示原型，从而判断该项目是否真正落地。

### 3. 研究内容

本项目的研究内容主要包括四部分。

第一，文本与材料整理。团队将以“王氏四种”为核心材料，优先整理与方法特征较鲜明的条目、札记和代表性案例，同时补录其所涉及的原始经典片段和引证材料。材料整理时将重点保留出处信息，尽量做到可以逐级回溯至卷次、篇章、条目或其他可识别位置。

第二，结构化标注与案例抽取。结合前期讨论，项目将“单条考据”作为基本工作单元，围绕问题触发点、主要方法、证据来源、过程说明、暂定结论等维度进行标注。对于当前阶段不宜过度细分的内容，先采用整体记录方式保存，避免为了形式完整而提前设计过重结构。

第三，知识库与演示系统实现。依据现有数据库设计方案，当前阶段采用“著作—文本片段—词条—案例—证据”五类核心对象组织数据，先完成录入、检索、展示的最小闭环。项目组已形成对应的字段规范与本地 Demo 原型，后续将在此基础上继续补充数据和完善交互，使申报期、中期和结项阶段都能有对应的可展示成果。

第四，方法归纳与学术分析。在案例累积到一定规模后，团队将对二王处理疑难时的常见路径进行归纳，例如改义、通假、改形、句法辨析、联绵词处理等，并观察不同书种、不同问题类型与不同证据使用方式之间的关联。这一部分既服务于平台展示，也服务于研究报告撰写。

第五，研究成果的前端化与答辩化表达。项目不把网站视为独立于研究之外的“包装层”，而是把它作为研究成果表达的一部分。当前原型已经区分首页展示与数据库结构说明页，后续将继续优化案例浏览、字段说明、检索入口和阶段性统计，使教师或评审在短时间内既能看到学术对象，也能看到团队对数据结构和实现路径的把握。对本科项目而言，这种“数据库后端 + 演示界面前端”的双重成果形态，有助于提高成果的可检验性和说服力。

### 4. 研究方法

本项目采用文献学方法与数字整理方法结合的方式推进。

1. 文献细读与个案分析。对代表性条目进行逐条研读，明确原文位置、讨论对象和判断逻辑，保证结构化结果不脱离具体文本。
2. 结构化标注。将单条考据拆分为若干稳定字段，形成可录入、可检索、可比对的数据单元。
3. 分类归纳。对不同问题类型、术语使用和证据形态进行整理，观察其共性与差异。
4. 数据建模。依据当前已形成的字段规范，优先建立“著作、文本片段、词条、案例、证据”五类核心对象之间的稳定关系，保证数据库结构与学术问题相匹配，而不是为了追求形式复杂度而过度拆分。
5. 原型开发。通过轻量数据库与本地 Web Demo 验证整理方案是否能够支撑检索和展示，并据此反向修正字段设计、录入流程和展示逻辑。
6. 人工复核优先。任何自动处理或程序辅助都只作为整理工具，不替代最终学术判断；涉及疑难归类和结论确认时，以人工核校为准。

### 5. 技术路线与实施步骤

结合当前项目基础，实施步骤拟分四阶段推进。

#### 阶段 A：数据准备与规则确定

1. 清理现有 Word、Markdown 等材料。
2. 统一案例记录格式、出处标记方式和术语口径。
3. 明确当前阶段采用的最小必要数据库结构。

#### 阶段 B：基础语料录入与检索原型

1. 录入首批代表性著作信息、文本片段和案例数据。
2. 建立词条、案例、证据之间的基本连接关系。
3. 完成知识库 Demo 中的基础检索、统计和案例浏览功能。
4. 区分首页展示、数据库架构说明和案例检索三个基本界面层次，确保研究内容与展示结构互相对应。

#### 阶段 C：案例扩充与方法归纳

1. 扩充案例数量，覆盖若干典型问题类型。
2. 对常见方法路径进行分类整理。
3. 结合数据库中已录入的案例、证据和术语，形成阶段性研究报告和标注规范说明。
4. 进一步优化网站展示，使案例、出处、证据和方法标签能够形成相互印证的阅读路径。

#### 阶段 D：探索性扩展

1. 在已有数据基础上试做更细关系或规则表示。
2. 讨论是否进入更复杂的关联结构或知识图谱表达。
3. 这一阶段仅作为探索性延伸，不设过高刚性指标。

### 6. 可行性分析

本项目具备较好的实施基础。

其一，材料边界清楚。项目以“王氏四种”为核心，不做无边界扩展，便于在一年周期内集中力量完成较高质量的专题整理。

其二，团队分工明确。学术整理、技术实现和数据处理已有对应成员承担，前期讨论也已经形成了较清楚的任务链条。

其三，已有原型基础。项目组已完成数据库字段规范、最小必要结构评估和本地 Demo 原型，原型中已包含基础案例数据、数据库结构说明和网页端展示逻辑。这意味着系统实现不是空想，而是已有可继续迭代的基础工程，也说明团队对“学术对象如何转化为可展示数据”已有初步实践。

其四，目标设置克制。项目不承诺当前无法证明的准确率、推荐效果或自动判读水平，不把“智能推理”写成必须完成的刚性成果，而是将重点放在文本整理、案例结构化、数据库搭建和工具原型建设上。这种安排更符合本科项目周期，也更有利于形成可验收成果。

### 7. 风险与应对

1. 文本质量不一。部分材料可能存在 OCR 误差或版本差异。对此，项目将优先保留出处信息和原文片段，逐步精修，不以一次录入即达到最终标准为前提。
2. 结论存在多解。古籍解释并非都能形成唯一答案。项目将通过“暂定结论”“待核状态”等方式记录不确定性，而不强行求单解。
3. 工作量可能超出预期。对此将坚持“先完成最小闭环，再考虑扩展”的原则，优先保证 A-C 阶段成果。

## 五、项目特色与创新之处

**【说明】** 包括如何突破现有研究难点，提出创新的理论视角或实践应用或技术改进等，以体现本研究独特的学术贡献与创新价值。限1000字，含标点符号。

本项目的特色与创新主要体现在以下几个方面。

第一，研究对象选择具有明确聚焦度。项目不泛泛而谈“古籍数字化”，而是围绕高邮二王这一方法特征鲜明、材料相对集中、学术影响明确的对象展开，能够在较清晰的边界内讨论传统考据方法如何被数字化表达。这样的对象选择既保证了学术价值，也保证了项目在一年期内具有可操作性。

第二，整理重点从“结果摘录”推进到“过程表达”。现有材料整理往往偏重收集结论，而本项目将“问题如何提出、证据如何组织、结论如何形成”视为同样重要的整理对象，通过案例、证据、术语和文本出处的联动，尽量保留二王考据的工作链条。这一点比单纯做全文库或词条库更能体现学术特色，因为它真正回应了训诂学研究中最难保存、却最有价值的部分。

第三，数据库设计与研究问题直接对应，而不是停留在技术口号层面。项目当前采用的最小必要结构，把著作、文本片段、词条、案例、证据作为核心对象，其重点不是追求表数多少，而是保证“二王说了什么、讨论了什么原文、如何展开论证、用了哪些证据”能够被清楚表达。对于训诂学专题数据库而言，这种以学术问题反推数据结构的做法，本身就是方法上的创新，因为它把“考据过程”而非“技术概念”置于系统设计中心。

第四，工程路径采取克制而可扩展的策略。项目不直接以“大规模知识图谱”作为起点，而是先构建最小必要知识库，完成可运行原型、网页展示和结构化案例，再根据数据积累讨论更复杂的关系扩展。这种做法既避免空泛承诺，也使创新建立在真实可落地的基础之上。对本科申报而言，这种“先做出可信原型，再逐步深化”的路径，本身比夸大远景更有说服力。

第五，项目兼具学术研究、成果展示与教学转化价值。对研究者而言，该资源有助于减少重复检索和材料分散带来的时间消耗；对教学而言，它能够把传统训诂学中不易显现的判断步骤转化为可展示、可比较的案例材料；对项目答辩与阶段验收而言，数据库与网站原型的结合又使成果不只停留在纸面叙述，而能以可操作界面体现研究进度和整理深度。这种“研究内容即展示内容”的一致性，也是本项目区别于单纯论文型选题的重要特点。

## 六、项目经费预算

**【说明】** 包括大致支出科目、金额、计算明细和必要性。限800字，含标点符号。

本项目坚持经费使用从简、围绕研究核心支出的原则，不购置与项目关联较弱的硬件设备。预算拟作如下安排：

1. 电子产品类：0元。项目不新购电脑、移动硬盘等通用设备，避免经费分散。
2. 信息服务类：1200元。主要用于云服务器、云存储、文档协同与按需 OCR 服务等基础信息服务，保障知识库原型运行、数据备份和多人协作。
3. 文献资料类：1800元。主要用于购置与项目直接相关的工具书、影印本、必要复印扫描费及馆藏资料调阅复制费，服务于案例核校与背景材料补充。
4. 差旅费类：1000元。主要用于在京文献查阅、馆藏核对和必要学术交流活动中的市内交通与相关支出，不作远距离调研铺张安排。
5. 组织类：600元。主要用于资料打印、文本装订、阶段性汇报材料制作和小范围讨论所需的基础物料。
6. 专家咨询及成果整理类：1200元。主要用于邀请相关方向教师或研究者进行阶段性指导，以及结项前成果整理和规范化修改。

以上合计5800元。该预算以文献、数据、平台与学术咨询为核心，不列与研究关联度较低的消费项，符合本项目“小而实”的实施原则。
